    Linked Open Data and INSPIRE

    • INSPIRE and LOD share a goal: add universal semantics to data
    • The approach is different

    XML+XSD+XSLT = RDF?

    • Inspire XSD's can be used to convert INSPIRE GML + ISO19139 to RDF
    • Not for as-is data and when the XSD's are extended locally

    Unique URI's

    • Don't use UUID identifiers, use URI's
    • A WFS getFeauture request, filtered by ID, could be a URI
    • The WFS server should support RDF or JSON-LD

    An alternative LOD approach

    • Build for Environment Canada Open Data
    • Publish data as JSON-LD in a WEB Accessible Folder
    • Each folder has datapackage-ld.json

    datapackage.jsonld

    • Provides metadata on the dataset
    • Provides links to broader/narrower
    • Adds a HTML representation of the folder contents

    Integration in Geonetwork Opensource

    • Validation of the uploaded data files (shape/xls/kml)
    • Workflow to manage metadata and data (ETL)
    • Simple UI to assign semantics to column names
    • Could be a first step to harmonisation (for as-is data)
